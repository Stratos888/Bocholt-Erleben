# Projekt: Bocholt erleben

## Zweck
Lokale Event-Plattform für Bocholt.
Fokus: Übersicht, Qualität, Vertrauen, kein Werbechaos.

## Feste Rahmenbedingungen
- Hosting: STRATO
- Source of Truth: GitHub Repository
- Deployment: statische Dateien
- Architektur: Vanilla HTML / CSS / JS
- PWA mit Service Worker
- Kein Backend (Stand jetzt)
- Keine Frameworks

## Arbeitsregeln (verbindlich)
- Immer nur **eine Datei gleichzeitig** ändern
- Immer **konsolidierte Stände**, keine Patch-Stapel
- UX & Klarheit vor Feature-Ausbau
- Cache & Service Worker immer mitdenken
- Änderungen müssen **sichtbar überprüfbar** sein

## Produktprinzipien
- Gleichwertige Darstellung aller Events
- Keine Werbung
- Keine bezahlten Hervorhebungen
- Vertrauen wichtiger als Monetarisierung

## Aktueller Status
- MVP live
- Fokus: UX-Optimierung & Stabilität
- Abo & Login noch nicht umgesetzt
